We can generally divide geoprocessing into 3 categories

  • geometric measurements (area and length)
  • creating new geometries
  • tests for spatial proximity or relationships

Geoprocessing functions do their calculations with map units.

If area or distance measurements are involed, use projected data.

Geometric measurements

Let’s compute the area of Yosemite. First we have to project the boundary into UTM.

epsg_utm11n <- 32611
yose_bnd_utm <- st_transform(yose_bnd, epsg_utm11n)
yose_area <- st_area(yose_bnd_utm)
yose_area
## 3019918537 [m^2]

st_area uses the units package to help convert between units.

library(units)

## View in square miles
set_units(yose_area, mi^2)

## View in hectacre
set_units(yose_area, ha)

Creating new geometries

Buffer

Let’s put a 1km buffer around each of the campsites.

# Project the points of interest to UTM
yose_poi_utm <- st_transform(yose_poi, epsg_utm11n)

## View the unique values in POITYPE
yose_poi_utm %>% st_drop_geometry() %>%  select(POITYPE) %>% distinct(POITYPE) %>% arrange(POITYPE)

## Pull out just the features which are campgrouds
yose_campgnd_utm <- yose_poi_utm %>% filter(POITYPE=="Campground")

## Create the 1km buffer
yose_campgnd_buff_utm <- st_buffer(yose_campgnd_utm, dist=1000)

## Plot
plot(st_geometry(yose_bnd_utm))
plot(st_geometry(yose_campgnd_buff_utm), add=TRUE)
