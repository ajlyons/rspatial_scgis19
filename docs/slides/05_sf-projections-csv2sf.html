Spatial Data Analysis with R

Working with sf Objects

Projections

Projections are particularly important whenever you want to:

  • overlay layers in a plot (R does not reproject on the fly)
  • use geometric measurements, like distance or area
  • do spatial queries or join layers

The more generic term for projections as is Coordinate Reference System (CRS).

CRS also includes unprojected geographic coordinates (longitude & latitude).

A CRS has three parts:

  1. a mathematical representation of the Earth (datum)
  2. a 2D to 3D transformation model (projected CRSâ€™s)
  3. an origin

CRS functions

All sf objects are able to store projection info.

st_read() imports the CRS info from standard GIS file formats

sf_crs() creates, views, or assigns the CRS

sf_transform() (re)projects sf objects into a different crs

Using Known CRSs

The sf package uses the PROJ.4 library for established CRS. It provides two ways to specify a known projection:

  • epsg numbers (recommended)
  • proj4 strings

The following are equivalent:

st_crs("+init=epsg:3857")
st_crs(3857)

If you happen to know the proj4string, you can use it also

st_crs("+proj=utm +zone=11")
## Coordinate Reference System:
##   EPSG: 32611 
##   proj4string: "+proj=utm +zone=11 +ellps=WGS84 +units=m +no_defs"

Common epsg numbers

Geographic CRSs

  1. Geographic, WGS84 (default for lon/lat)
  2. Geographic, NAD83 (USA Fed agencies like Census)

Projected CRSs

  1. USA Contiguous Albers Equal Area Conic
  2. CA ALbers Equal Area
  3. UTM Zone 10, WGS84 (Northern Cal)
  4. UTM Zone 11, WGS84 (Southern Cal)
  5. Web Mercator (web maps)

The one-stop shop for finding proj4 strings and epsg numbers is http://www.spatialreference.org. (Google usually works also).

Reference code for your convenience:

epsg_geo <- 4326
epsg_utm10n <- 32610
epsg_utm11n <- 32611
epsg_webmerc <- 3857
epsg_caalbers <- 3310

View or Assign the CRS

View the CRS of a sf object:

st_crs(yose_trails)
## Coordinate Reference System:
##   EPSG: 26911 
##   proj4string: "+proj=utm +zone=11 +ellps=GRS80 +towgs84=0,0,0,0,0,0,0 +units=m +no_defs"

Occasionally, you have a sf object that doesnâ€™t have a CRS recorded. This is most common when you are importing a CSV file and are turning it into a sf object. In thse cases, you can use st_crs() to assign the CRS.

Note: this is not the same as projecting data. Youâ€™re simply telling it the CRS.

You can project data from one CRS to another with:
